# Membership App Agent Guide

This is a premium membership application.

## Main expectations
- Preserve existing business logic.
- Preserve auth, payments, and routing unless explicitly asked to change them.
- Make visual improvements without breaking functionality.
- Prefer surgical edits.

## UI direction
- Premium, modern, polished
- Mobile-first
- Subtle 3D depth effects only where useful
- Clear status cards, announcements, events, dues, and member actions

## Engineering rules
- React + TypeScript safe code only
- Reuse existing patterns
- Avoid unnecessary dependencies
- Keep components readable and production-ready
